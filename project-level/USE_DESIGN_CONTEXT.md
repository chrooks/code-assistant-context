## Design & UI/UX Guidelines

When making any visual, frontend, or UI/UX changes, you **must** consult both files below.

- **`/context/design-principles.md`** — S-tier SaaS design checklist covering philosophy, design system, layout, interactions, and CSS architecture. Use it to validate quality.
- **`/context/style-guide.md`** — Project-specific style guide with concrete design tokens (colors, typography, spacing), component specs, accessibility rules, and microcopy standards. This is the source of truth for implementation.

If a decision isn't covered, match the closest existing pattern rather than inventing a new one.
